Reference implementation

Compute it yourself.

The CAI is only as trustworthy as it is reproducible. The reference implementation is open source under Apache-2.0, so you can run the scorer on your own evidence — or anyone else's — and get the same number.

How it works

The standard splits cleanly in two. Measuring a codebase into an evidence bundle (the per-lens scores + their weights) is the analyzer's job. Scoring that bundle — folding the lenses worst-first into the 0–100 CAI and banding it — is open, deterministic, and what this reference implementation does. So you can recompute and verify any published number from its evidence, no engine required.

# build the reference scorer (C#, Apache-2.0)
git clone https://github.com/CanineCC/CAI
cd CAI/scorer

# score an evidence bundle
dotnet run --project Cai.Cli -- score examples/evidence.sample.json

# → CAI 72 (Strong)  ·  rubric rubric-2026.06.0
#     securityCompliance  58.0   w0.240   13.92   ← worst lens, most weight
#     maturity            65.0   w0.200   13.00
#     ...   (the full lens fold is printed, nothing hidden)

The headline is Σ(lensScore × owaWeight) — an ordered-weighted average that puts the most weight on your weakest lens, so a real problem can't be averaged away behind strengths. The weights are published in the evidence, so the fold is reproducible by anyone. Nothing about the number is hidden, and you never have to take it on trust.

Why open

No black box

A score you cannot reproduce is an opinion. Publishing the algorithm turns "trust us" into "check it."

One vocabulary

The lenses, the rubric, the bands — the same terms a survey, a contract, and a badge all reference. Define once, cite everywhere.

Method open, judgment sold

The standard is free and open. The independent, signed survey — the deductions and what to do — is the service. That's Watchdog.

Have a survey with its evidence? Run the scorer against it → /verify and confirm the published number for yourself.
