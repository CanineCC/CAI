The open standard

An open, reproducible 0–100 standard for the health of a codebase.

Same code in, same score out — a measurement anyone can verify. The Codebase Assurance Index (CAI) reduces a whole C#/.NET codebase to one deterministic number across ten lenses, computed from a frozen, versioned rubric. No opinions, no black box.

Open · Verifiable · Referenced

Open

The full algorithm is public and versioned — the ten lenses, the frozen rubric, the bands. A reference implementation ships under Apache-2.0. Nothing about the number is hidden.

Verifiable

The score is deterministic. Given the same evidence and rubric version, the scorer returns the same number — and you can run it yourself. A claim is only as good as its reproducibility.

Referenced

Surveys are signed and published to a public registry. A CAI on a contract, a badge, or a due-diligence pack points back to a record anyone can check.

The firewall: method open, judgment independent.

The standard (the how it's measured) is open so anyone can reproduce a number. The judgment — an independent, signed survey with the deductions and what to do — is a service. The line between them is the firewall, and it's deliberate.

Ten deterministic lenses.

The CAI is composed of dimensions grouped by lens — each lens a measurement of the code as it stands, never a judgement of the team. Core lenses are always on; model-aware lenses light up only when the architecture calls for them.

A public record, not a logo wall.

Every signed survey can be published, in full, to the registry — the evidence and the number, open to inspection. Falsifiable by design: anyone can re-run the scorer against the evidence and get the same result.
